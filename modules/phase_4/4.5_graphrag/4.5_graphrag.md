## When to Do This Module

Do this module if, after Module 4.4, you observe **multi-hop queries failing** — queries that require connecting facts across two or more separate documents to produce a correct answer.

Examples that top-k retrieval struggles with:
- *"Does the CSV export bug affect DeskMate Pro users on the Enterprise SLA?"* — requires linking `technical_bugs.md` (the bug) + `billing_refunds.md` (plan tiers) + `outage_status.md` (SLA commitments).
- *"If I'm on the plan with the highest SLA, what's my uptime guarantee and where do I report outages?"* — two documents, one answer.
- *"Which integration requires admin access in both systems, and what plan is it on?"* — scattered across `getting_started.md`.

**If your Module 4.4 hit-rate@3 is ≥ 0.90 and multi-hop queries represent < 5% of your ticket volume, skip this module.**

---

## What GraphRAG Does

Standard RAG retrieves the top-k most similar chunks independently. GraphRAG adds a **knowledge graph layer** — entities and relationships extracted from your documents — so the retriever can follow edges between concepts rather than doing independent nearest-neighbour lookups.

```
Standard RAG:
  query → embed → find nearest chunks → generate

GraphRAG:
  query → extract entities → traverse knowledge graph → find connected subgraph
        → retrieve chunk(s) that cover the subgraph → generate
```

---

## Knowledge Graph Structure

For DeskMate, the graph would contain:

**Nodes (entity types):**
- Products: `DeskMate Pro`, `DeskMate Enterprise`, `DeskMate Mobile`
- Plans: `Pro Plan`, `Enterprise Plan`, `Free Plan`
- Features: `CSV Export`, `2FA`, `SSO`, `API Access`, `CSV Export`
- Issues: `ERR-500 CSV Bug`, `iOS 17 Crash`
- Policies: `30-day Refund`, `99.9% SLA`, `99.99% SLA`

**Edges (relationship types):**
- `AFFECTS`: `ERR-500 CSV Bug` → `DeskMate Pro`
- `INCLUDED_IN`: `SSO` → `Enterprise Plan`
- `GUARANTEES`: `Enterprise Plan` → `99.99% SLA`
- `DOCUMENTED_IN`: `ERR-500 CSV Bug` → `technical_bugs.md::chunk_1`
- `FIXED_IN`: `ERR-500 CSV Bug` → `version 4.3.0`

---

## How GraphRAG Answers Multi-Hop Queries

Query: *"Does the CSV export bug affect Enterprise SLA customers?"*

1. Entity extraction: `{CSV export bug, Enterprise SLA}`
2. Graph traversal:
   - `ERR-500 CSV Bug` → `AFFECTS` → `DeskMate Pro`
   - `Enterprise Plan` → `GUARANTEES` → `99.99% SLA`
   - `DeskMate Pro` ≠ `Enterprise Plan` → add note
3. Retrieve chunks linked to traversed nodes:
   - `technical_bugs.md::chunk_1` (bug description)
   - `outage_status.md::chunk_2` (SLA commitments)
4. Assemble multi-document context and generate.

Standard top-k retrieval would likely retrieve one or the other document — not both — because "CSV export bug" and "Enterprise SLA" are in different semantic neighbourhoods.

## Implementation Sketch (Microsoft GraphRAG)

The `graphrag` library by Microsoft automates entity extraction and graph construction from a document corpus:

```python
# 1. Initialise
import graphrag.api as api

# 2. Index (extract entities + build graph from docs)
# This is expensive — runs an LLM over every chunk to extract entities
await api.build_index(root_dir="data/graphrag_workspace/")

# 3. Query (local = within-community, global = across the full graph)
result = await api.local_search(
    root_dir="data/graphrag_workspace/",
    query="Does the CSV export bug affect Enterprise SLA customers?",
)
print(result.response)
```

**Cost warning:** GraphRAG indexing calls an LLM (typically GPT-4 or Claude) for every chunk to extract entities and relationships. For 100 chunks this costs ~$0.50–2.00 depending on the model. Re-indexing is required whenever documents change.

## GraphRAG vs Vanilla RAG Trade-Off

| Dimension | Vanilla RAG (4.2–4.4) | GraphRAG |
|---|---|---|
| Setup cost | Low (embed + index) | High (LLM-based entity extraction) |
| Per-query cost | Low (vector search) | Higher (graph traversal + retrieval) |
| Re-indexing on doc update | Fast (re-embed chunks) | Slow (re-extract entities) |
| Single-doc factual Q&A | ✅ Excellent | ✅ Excellent |
| Multi-hop / relational Q&A | ❌ Poor | ✅ Good |
| Explainability | Chunk citations | Entity + relationship path |
| Latency | Low (~100ms) | Higher (~500ms–2s) |

For DeskMate's primary use case (single-ticket support queries), vanilla RAG covers > 95% of cases. GraphRAG adds value for complex product/policy/plan intersection queries.
